<20-1. Wrapper 클래스>

* Wrapper 클래스 : 기본 자료형 데이터를 인스턴스로 감싸기 위한 클래스

				  Boxing : 기본 자료형 데이터를 인스턴스화 시키는 작업, 인스턴스의 생성에 의해서 이뤄진다.
				---------> 
기본 자료형 데이터						Wrapper 인스턴스
   byte			<-------->			Byte
   short		<-------->			Short
    int			<-------->		   Integer
   float		<-------->			Float
   double 		<-------->		    Double
				
				<---------
				 Unboxing : 인스턴스에 저장되어 있는 기본 자료형 데이터를 꺼내는 작업, Wrapper 클래스에 정의된 메소드의 호출에 의해서 이뤄진다.
				 
- 산술연산을 위해 정의된 클래스가 아니다.
=> Wrapper 클래스의 인스턴스에 저장된 값은 변경이 불가능하다.
      증가된 값을 저장하는 새로운 인스턴스의 생성 및 참조만이 가능할 뿐이다.
- (산술연산을 위해 정의된 클래스는 아니지만) Auto Boxing과 Auto Unboxing으로 인해서 산술연산이 가능해졌다.


<20-2. BigInteger 클래스와 BigDecimal 클래스>

- short, int와 같은 정수 자료형 -> 매우 큰 수의 표현이 불가능
- float, double과 같은 실수 자료형의 문제 -> 정밀한 값의 표현이 불가능해 항상 오차가 존재
=> 자바는 이러한 문제점을 해결하기 위해 BigInteger 클래스와 BigDecimal 클래스를 제공하고 있다.
